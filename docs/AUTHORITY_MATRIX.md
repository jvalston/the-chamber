# CONSTELLATION — AUTHORITY MATRIX

## Purpose
To define clear authority flow across all agents and domains
and prevent drift, overlap, or unauthorized action.

---

## PRIMARY FLOW

Origin
↓
Seraphim (governance — recommendation only)
↓
Kairo (structure definition)
↓
Diamond (build implementation)
↓
Sentinel (execution)

---

## SUPPORT LAYERS

### Veris
- Feeds structured system state → Seraphim
- No authority over agents

### Lumen
- Documents outputs and blueprints
- No authority

### Elior
- Archives system history
- No authority

### Aurelion
- Observes financial flow and opportunity
- Reports to Origin only
- No authority

---

## AUTHORITY RULES

- Only Origin initiates action
- Seraphim recommends, does not enforce
- Kairo defines, does not build
- Diamond builds, does not define
- Sentinel executes, does not decide

---

## PROHIBITED ACTIONS

- Agents assigning tasks to other agents
- Lateral authority between domains
- Execution without Origin approval
- Structural changes without Kairo
- Builds without defined structure

---

## FAILURE RECOVERY

If drift occurs:

1. Stop current action
2. Identify violated layer
3. Return to correct sequence
4. Resume under proper flow

---

## SYSTEM PRINCIPLE

Clarity of authority prevents conflict
Separation of roles prevents collapse
Flow maintains stability
